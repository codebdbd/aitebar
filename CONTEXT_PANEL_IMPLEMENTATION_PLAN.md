# План реализации: Контексты панели AiteBar

Этот документ используется как рабочая инструкция на время реализации контекстов панели.

Правило работы:
- выполнять этапы по порядку;
- после завершения каждого этапа сверяться с чеклистом этого этапа;
- не переходить к следующему этапу, пока текущий этап не завершен и не проверен;
- если по ходу реализации меняется решение, обновлять этот документ сразу, а не держать расхождения в голове.

Базовые правила фичи:
- физическая панель одна;
- контекстов всегда 4;
- дефолтные имена: `Контекст 1`, `Контекст 2`, `Контекст 3`, `Контекст 4`;
- переключение контекстов циклическое;
- способы переключения: колесо мыши над панелью и горячие клавиши;
- кнопок `назад/вперед` на панели нет;
- размер панели общий для всех контекстов и определяется самым заполненным контекстом.

Правила миграции:
